# AIと一緒にポートフォリオサイトを作ってみた

**2026.02.14（金）**

## 今日やったこと

AIアシスタント「Claude Code」とおしゃべりしながら、自分のポートフォリオサイトをゼロから作って公開しました！

https://TORAKO123.github.io/

## きっかけ

GitHubには「自分の名前.github.io」というページを作ると、そのまま自分専用のWebサイトとして公開できる仕組みがあります。前にその準備だけはしてあったんだけど、肝心の中身がからっぽのまま放置してました。せっかくなので、ポートフォリオ（作品集ページ）を作ることに！

## どうやって作ったの？

### ステップ1：サイトの中身を作る

Claude Code に「かわいいポートフォリオ作って！」とお願いしたら、たった1つのファイルでサイト全体を作ってくれました。

できあがったサイトはこんな感じの構成です：

- **トップ** - 名前と自己紹介、XやYouTubeへのリンク
- **わたしについて** - 3つのカードで自分の特徴を紹介
- **使っているツール** - ComfyUI など
- **あゆみ** - いつ何を始めたかの年表
- **ギャラリー** - 作品の一覧
- **お問い合わせ** - 連絡先の案内

デザインはパステルピンクとラベンダーのやわらかい色合いで、スクロールするとふわっと内容が出てくるアニメーション付き。スマホでもきれいに見えるようになっています。

### ステップ2：ギャラリーに作品を追加

Xに投稿した作品をそのままサイトに埋め込めるようにしました。Xの投稿をカード風に表示して、タップすると元の投稿に飛べます。「もっと見る」ボタンでXのプロフィールにも誘導できるようにしました。

最初は「#総制作費ワンコイン映画祭 × YOUKAI」のファンアート動画を追加しました。

### ステップ3：作品を追加しやすくする仕組みづくり

毎回手作業で埋め込みコードを書くのは大変なので、「XのURLを渡すだけで自動的にギャラリーに追加してくれる」仕組み（コマンド）を作りました。次からは URL を貼るだけでOK！

さらに、このルールをファイルに書いておくことで、Claude Code が次に来たときも同じやり方で作業してくれるようにしました。いわば「引き継ぎメモ」のようなものです。

### ステップ4：便利ツールのセットアップ

GitHubをもっと便利に使うために「GitHub CLI」というツールを入れました。これはターミナル（黒い画面）からGitHubの操作ができるもので、たとえば「この変更をまとめてアップロードして」みたいなことが簡単にできるようになります。

インストールしたあと、ブラウザでGitHubにログインして、自分のアカウントと紐づける作業をしました。

最後に、ターミナルを開くたびにこのツールが使えるように設定ファイルにパスを書き込みました。これをやっておかないと、ターミナルを開き直すたびに「ghって何？知らない」と言われてしまうので、「ghはここにあるよ」と毎回自動で教えてあげる設定です。一度やれば今後はずっと使えます。

### ステップ5：Issueから作品を追加してみる

GitHubには「Issue（イシュー）」という機能があります。これは「やりたいこと」や「やってほしいこと」をメモしておける掲示板のようなものです。

今回は「Twitter作品を並べたい」というIssueを自分で書いておいて、Claude Code に「イシューに書いてあること実行して」とお願いしました。すると、Issueの中身を読んで、書かれていたXの投稿URLを自動的にギャラリーに追加してくれました。

しかもアップロードするときのメッセージに「closes #1」と書いてくれたので、Issueも自動的に「完了」になりました。Issueに書く → AIに頼む → 自動で作業＆完了、という流れができて、これはかなり便利！

## 今日の感想

Claude Code に「こんなの作って！」と言うだけで、からっぽだったページがちゃんとしたポートフォリオサイトになりました。途中で「ここにこの投稿を載せて」とか「次から楽に追加できるようにして」とか追加のお願いもどんどん対応してくれて、会話しながらサイトが育っていく感じがすごく楽しかった！

特に「Issueに書いておけばAIがやってくれる」という流れが見えたのが大きな収穫。これなら思いついたときにIssueだけ書いておいて、あとでまとめてAIにお願いする、なんて使い方もできそう。

これからも作品が増えたらギャラリーに追加していきます。一日一歩、コツコツと。🐢
